# Security policy

Security fixes are provided for the latest released version. Please report a
suspected vulnerability privately to the project maintainer before opening a
public issue. Include the affected version, reproduction steps, impact, and any
suggested mitigation. Do not include real user data, credentials, or private
uploads.

For internet-facing deployments, use HTTPS, set an admin password, keep the
container current, and expose VortNotes through a trusted reverse proxy or VPN.
Back up `/data` before upgrades. Uploaded content is user-controlled and must
not be included in public images, source archives, or bug reports.
